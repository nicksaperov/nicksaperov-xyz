MODULE 26: Cross-Language Asynchrony & Race Conditions

Core Logic: EZKL operations (like downloading the massive Structured Reference String (SRS) parameter file) trigger background asynchronous network calls via PyO3 Rust bindings. These bindings return foreign Rust Future objects to the Python layer.

26.1 Awaitable Object Inspection

Standard Python asyncio.iscoroutine() checks evaluate Rust Futures as False. This type-checking blindspot causes the Python event loop to sprint ahead synchronously and attempt to execute the cryptographic Setup phase before the SRS file has physically finished downloading to the disk, resulting in a fatal crash.

The compiler utilizes inspect.isawaitable() to dynamically recognize and safely block on cross-language asynchronous primitives, guaranteeing linear state execution.
